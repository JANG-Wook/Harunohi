# 02. 기술 스택

`infobank-kakao-open-builder` 의 검증된 스택을 기반으로 하되, RAG·임베드 위젯 같은 신규 요구를 위한 추가 컴포넌트를 더한다.

## 1. 채택 스택

| 영역 | 기술 | 근거 |
|------|------|------|
| **Backend** | Spring Boot 3.5 / Java 21 | 인포뱅크 팀 보유 역량, 레퍼런스 코드 패턴 재사용 |
| **Frontend (빌더 콘솔)** | React 19 + Vite | 동일 레퍼런스, HailMary 자산도 React 기반 |
| **의도 추론 서비스** | Python FastAPI | 임베딩·머신러닝 생태계 우위 |
| **Relational DB** | MySQL 8.x | 레퍼런스 일치, 운영 인프라 보유 |
| **Vector DB** | Qdrant (예정) | RAG용. MySQL은 벡터 미지원, 별도 분리 권장 |
| **임베딩 모델** | `intfloat/multilingual-e5-large-instruct` (Hugging Face TEI) | 한국어 지원, 레퍼런스에서 검증됨 |
| **LLM Provider** | Claude API (1차), OpenAI 호환 인터페이스 (확장) | RAG 응답 생성, LLM 의도 분류 |
| **임베드 위젯** | Vanilla JS + iframe | CSS 격리, 보안, 호환성 |

## 2. 서비스 분리

레퍼런스의 3계층을 그대로 따른다.

```
┌──────────────────────────────────────────┐
│ 빌더 콘솔 (React + Vite)                  │
└────────────────┬─────────────────────────┘
                 │ REST
┌────────────────▼─────────────────────────┐
│ Backend API (Spring Boot)                 │
│  - 봇 정의 / 버전 / 배포 / 인증 / 세션    │
└────┬────────────────────┬─────────────────┘
     │                    │
     ▼                    ▼
┌──────────────┐  ┌──────────────────────┐
│ MySQL        │  │ 의도 추론 (FastAPI)  │
│              │  │  - 임베딩 매칭        │
│              │  │  - LLM 분류 (옵션)    │
└──────────────┘  └──────────┬───────────┘
                             │
                             ▼
                  ┌──────────────────────┐
                  │ Qdrant (Vector DB)   │
                  │  - 문서 청크 벡터     │
                  │  - 의도 예시 벡터     │
                  └──────────────────────┘

     ┌────────────────────────────────┐
     │ 임베드 위젯 (Vanilla JS)        │
     │  ← 고객사 웹/앱에 <script> 삽입 │
     └────────────────┬───────────────┘
                      │ WebSocket / SSE
                      ▼
              ┌──────────────┐
              │ Backend API  │
              │ (대화 엔진)  │
              └──────────────┘
```

## 3. RAG 파이프라인

```
[문서 업로드 (.pdf, .docx, .txt)]
        │
        ▼
[텍스트 추출 + 청킹]   ← 청크 단위로 자름 (예: 500자)
        │
        ▼
[임베딩 생성 (TEI)]    ← 각 청크를 벡터로 변환
        │
        ▼
[Qdrant 저장]          ← 청크 메타데이터는 MySQL, 벡터는 Qdrant
        │
        ▼
   ───────────────────────────────────
        │
[사용자 질문]
        │
        ▼
[질문 임베딩 → Qdrant 검색]  ← 상위 N개 청크 검색
        │
        ▼
[LLM 호출 (Claude API)]      ← 청크 + 질문을 프롬프트로 전달
        │
        ▼
[자연어 답변 → 위젯 출력]
```

## 4. 열린 결정 사항

| # | 항목 | 후보 | 결정 시점 |
|---|------|------|----------|
| 1 | Vector DB | Qdrant / Pinecone / Weaviate | PoC 직전 |
| 2 | LLM Provider 추상화 | 단일 (Claude만) / 멀티 프로바이더 | MVP 중반 |
| 3 | 청킹 전략 | 고정 길이 / 문장 단위 / 시맨틱 | PoC에서 실험 |
| 4 | 임베드 위젯 격리 방식 | iframe / Shadow DOM | 위젯 개발 착수 시 |
| 5 | 인증 방식 | 세션 쿠키 (레퍼런스 따라감) / JWT | MVP 초반 |

## 5. 인프라 / 운영 (Phase 별)

- **MVP** — 단일 클라우드 인스턴스, MySQL/Qdrant 컨테이너로 띄움
- **Phase 2** — 멀티테넌시 강화, CDN으로 위젯 배포, 모니터링 추가
- **Phase 3** — 엔터프라이즈 배포 자동화 (Helm chart 등)

## 6. 레퍼런스에 없는, 우리가 새로 만들어야 하는 것

`infobank-kakao-open-builder` 에 없는 컴포넌트.

- **벡터 DB 연동** (Qdrant)
- **RAG 파이프라인** (문서 업로드, 청킹, 임베딩 저장, 검색, LLM 호출)
- **LLM 응답 생성기** (Claude API 클라이언트)
- **임베드 위젯** (`<script>` 한 줄로 삽입 가능한 chat widget)
- **공개 런타임 API** (위젯 ↔ 백엔드 대화 프로토콜)
