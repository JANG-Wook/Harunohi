# 05. 빌더 UI 와이어프레임

HailMary 디자인 시스템(`tokens.css` + 39개 컴포넌트)을 그대로 활용한다.
ASCII 다이어그램은 화면 구성을, 표는 사용할 디자인 토큰·컴포넌트를 정의한다.

## 0. 공통 디자인 원칙

### 0.1 글로벌 토큰 매핑

| 영역 | 토큰 |
|------|------|
| 메인 배경 | `--color-bg-normal` (#FFFFFF) |
| 보조 배경 (사이드바, 카드 외곽) | `--color-bg-normal-alternative` (#F7F7F8) |
| 본문 텍스트 | `--color-label-normal` (#171719) |
| 보조 텍스트 | `--color-label-alternative` |
| 비활성 텍스트/placeholder | `--color-label-assistive` |
| 경계선 | `--color-line-solid-normal` (#E1E2E4) |
| 강조 색 / CTA | `--color-primary-normal` (#0066FF) |
| 성공 | `--color-status-positive` (#00BF40) |
| 경고 | `--color-status-cautionary` (#FF9200) |
| 에러 | `--color-status-negative` (#FF4242) |

### 0.2 타이포그래피 매핑

| 용도 | variant |
|------|---------|
| 페이지 타이틀 | `title-2` (28px / Bold) |
| 섹션 헤딩 | `heading-2` (20px / Semibold) |
| 카드 타이틀 | `headline-2` (17px / Semibold) |
| 본문 | `body-1` (16px / Regular) |
| 라벨/입력 라벨 | `label-1` (14px / Medium) |
| 캡션/메타 | `caption-1` (12px / Regular) |

### 0.3 스페이싱

| 의미 | 값 |
|------|-----|
| 컴포넌트 내부 패딩 | `--spacing-12`, `--spacing-16` |
| 컴포넌트 간격 | `--spacing-16`, `--spacing-24` |
| 섹션 간격 | `--spacing-32`, `--spacing-40` |
| 페이지 좌우 패딩 | `--spacing-24` (Desktop), `--spacing-20` (Mobile) |

## 4. 봇 작업 화면 — 글로벌 레이아웃

봇을 클릭해 들어오면 **봇 작업 화면**으로 진입. 봇 전용 사이드바·탑바가 콘솔 위에 덮인다.

```
┌──────────────────────────────────────────────────────────────────────────┐
│ ← AS 신청봇                  [개요] [FAQ] [기능] [지식베이스] [설정]      │
│                              [💾 저장] [▶ 시뮬레이터] [🚀 발행 v3 →]      │
├──────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│                  탭별 콘텐츠                                              │
│                                                                          │
└──────────────────────────────────────────────────────────────────────────┘
탑바 64px / 탭이 핵심 네비게이션
```

### 사용 컴포넌트

| 영역 | 컴포넌트 |
|------|---------|
| 뒤로가기 + 봇 이름 | `IconButton` (chevronLeft) + `Typography` (heading-2) |
| 모드 탭 | `Tab` (size=medium, resize=hug) |
| 저장 | `Button` (variant=outlined, color=assistive, leadingIcon=save) |
| 시뮬레이터 토글 | `Button` (variant=outlined, leadingIcon=play) |
| 발행 | `Button` (variant=solid, color=primary) |

## 5. FAQ 모드

질문/답변 페어를 테이블 + 인스펙터로 편집. **캔버스 사용 안 함.**

```
┌──────────────────────────────────────────────────────────────────────────┐
│  ← AS 신청봇    [개요] [●FAQ] [기능] [지식베이스] [설정]   ...   [발행]   │
├────────────────────────┬────────────────────────────┬────────────────────┤
│                        │                            │                    │
│  카테고리                │  Q. 환불은 어떻게 하나요?    │ ▶ 시뮬레이터        │
│  ─────────             │  ─────────────────────     │                    │
│  📁 환불정책 (12)       │                            │  ┌──────────────┐  │
│  📁 배송문의 (8)        │  📝 응답 모드              │  │ ChatRoom 컴포넌트  │
│  📁 결제관련 (5)        │  ○ 정적 메시지              │  │              │  │
│  + 카테고리 추가         │  ● RAG 응답                │  │              │  │
│                        │                            │  │              │  │
│  ─────────             │  📚 참조 KB                │  │              │  │
│                        │  [정책 문서 KB        ▾]    │  │              │  │
│  Q&A 페어 (25)          │                            │  │              │  │
│  ─────────             │  💬 트리거 발화 예시        │  │              │  │
│  ✓ 환불은 어떻게...      │  • 환불하고 싶어요          │  │              │  │
│  ○ 배송 얼마나...        │  • 반품 가능한가요?          │  │              │  │
│  ○ 결제 취소...          │  + 예시 추가                │  │              │  │
│  ...                   │                            │  └──────────────┘  │
│                        │  🤖 답변 미리보기           │                    │
│  + Q&A 추가             │  (KB에서 검색된 컨텍스트     │  [메시지 입력...]   │
│                        │   기반 답변이 표시됨)        │                    │
└────────────────────────┴────────────────────────────┴────────────────────┘
   사이드바 240px         메인 영역 (가변)              시뮬레이터 360px
```

### 사용 컴포넌트

| 영역 | 컴포넌트 |
|------|---------|
| 카테고리 트리 | 커스텀 ListCell (HailMary `ListCell` 활용) |
| Q&A 리스트 | `ListCard` 또는 ListCell + `Checkbox` |
| 응답 모드 선택 | `Radio` (또는 `SegmentedControl`) |
| KB 선택 | `Select` (render=text, leadingIcon=folder) |
| 발화 예시 | `Chip` (variant=outlined, trailingContent=x) + 추가 입력 |
| 시뮬레이터 | `ChatRoom` (헤더, 메시지, 입력 통합) |

## 6. 기능 시나리오 모드 (캔버스)

```
┌──────────────────────────────────────────────────────────────────────────┐
│ ← AS 신청봇   [개요] [FAQ] [●기능] [지식베이스] [설정]   ...    [발행]    │
├──────────┬───────────────────────────────────────┬───────────────────────┤
│          │                                       │                       │
│ 노드 팔레트 │            캔버스                      │   인스펙터            │
│          │                                       │                       │
│ 📨 메시지  │   ┌─────────┐                          │  선택된 노드           │
│ ⌨ 입력    │   │ 시작     │                          │  ─────────           │
│ ☑ 선택지  │   └────┬────┘                          │  타입: 메시지          │
│ 📅 날짜   │        │                              │                       │
│ 🤖 RAG    │        ▼                              │  이름                  │
│ 🔌 API    │   ┌─────────────────┐                  │  [인사 메시지   ]      │
│ ⤴ 분기   │   │ 📨 인사메시지     │                  │                       │
│ ⛳ 종료   │   │ "AS 신청을 도와    │                  │  메시지 내용 *         │
│          │   │  드리겠습니다"     │                  │  [Textarea...]        │
│          │   └────┬────────────┘                  │                       │
│ ─────    │        │                              │  컴포넌트 추가          │
│ 변수      │        ▼                              │  [+ 카드] [+ 캐러셀]   │
│ • 제품종류 │   ┌─────────────────┐                  │                       │
│ • 고객명   │   │ ⌨ 고객명 입력     │                  │  ─────                │
│ • 전화번호 │   │ → 변수: 고객명    │                  │  엣지: 다음 노드 →    │
│          │   └────┬────────────┘                  │  • 고객명 입력         │
│          │        │                              │                       │
│          │        ▼ ...                          │                       │
│          │                              [+] [-] [⛶]  │  [🗑 노드 삭제]      │
└──────────┴───────────────────────────────────────┴───────────────────────┘
   220px              가변 (zoom in/out + minimap)        320px

   [▶ 시뮬레이터 토글] → 우측에 시뮬레이터 패널 오버레이로 추가됨 (인스펙터 위)
```

### 사용 컴포넌트

| 영역 | 컴포넌트 |
|------|---------|
| 노드 팔레트 | 커스텀 (드래그 가능한 카드. `Icon` + `Typography` label-1) |
| 노드 (캔버스) | 커스텀 React Flow + 토큰 적용 (shadow-small, radius 8px, primary border on select) |
| 인스펙터 입력 | `Textfield`, `Textarea`, `Select`, `Switch`, `SegmentedControl` |
| 컴포넌트 추가 | `Button` (outlined, leadingIcon=plus, size=small) |
| 노드 삭제 | `Button` (variant=outlined, color=negative) |
| 줌 컨트롤 | `IconButton` (Background) × 3 |

### 노드 비주얼 사양

```
┌─────────────────────────┐  ← shadow-small, radius 8px
│ 📨 메시지                │  ← 헤더: 아이콘 + 타입명 (caption-1)
│ ─────                   │
│ "AS 신청을 도와드리겠     │  ← 본문: 노드 요약 (label-2, 2줄 truncate)
│  습니다"                 │
└─────────────────────────┘
선택 시: border 2px solid --color-primary-normal
시작 노드: 좌상단 모서리에 ▶ 표시
```

## 7. 시뮬레이터 패널 (공통)

FAQ 모드·기능 모드 양쪽에 동일하게 우측 패널로 표시. **HailMary `ChatRoom` 컴포넌트를 그대로 활용.**

```
┌──────────────────────────────┐
│ ▶ 시뮬레이터              ✕  │
│ v3 (편집 중)  [🔄 초기화]   │
├──────────────────────────────┤
│                              │
│  🤖 안녕하세요!               │
│     AS 신청을 도와드리겠      │
│     습니다                   │
│                              │
│  🤖 어떤 제품인가요?          │
│     [냉장고] [세탁기] [TV]    │
│                              │
│                  냉장고 👤   │
│                              │
│  🤖 성함을 알려주세요         │
│                              │
├──────────────────────────────┤
│ [메시지 입력...]    [전송]    │
├──────────────────────────────┤
│ 🐞 디버그 (펼치기)             │
│ 현재 노드: 고객명_입력          │
│ 변수: { 제품종류: "냉장고" }   │
└──────────────────────────────┘
```

### 사용 컴포넌트

| 영역 | 컴포넌트 |
|------|---------|
| 시뮬레이터 컨테이너 | `ChatRoom` |
| 메시지 표시 | ChatRoom 내장 + 모든 HailMary 컴포넌트 (Card, Carousel, Calendar 등) |
| 디버그 패널 | 커스텀 펼침형 (`Icon` + 토글) |
| 초기화 | `Button` (size=small, outlined) |

## 13. 반응형 / 디바이스 대응

| 화면 | Desktop (≥1200) | Tablet (768~1199) | Mobile (<768) |
|------|----------------|-------------------|---------------|
| 콘솔 (빌더) | 풀 레이아웃 | 사이드바 축소 | **미지원** (MVP 제외) |

| 시뮬레이터 | 우측 360px 패널 | 모달로 전환 | - |

> 빌더는 데스크탑 우선. 위젯은 데스크탑·모바일 모두 지원.

## 15. 우선 구현 순서 (MVP)

| Phase | 화면 |
|-------|------|
| **W1** | 로그인 · 글로벌 레이아웃 · 대시보드 |
| **W2** | 봇 생성 · 봇 개요 · FAQ 모드 (기본) |
| **W3** | 시뮬레이터 패널 · 지식베이스 + 문서 업로드 |
| **W4** | 기능 시나리오 모드 (캔버스, 노드 5종) |
| **W5** | 발행 · 버전 관리 · 변수 관리 |
| **W6** | 임베드 위젯 · 종단 대화 흐름 |

각 주차는 빌더 사용자가 "끝에서 끝까지" 경험할 수 있는 수직 슬라이스 단위.
